TẦNG 3: DATA LAYER

Database / Data Storage / Persistence Layer
PORT: 5432

🐘 PostgreSQL Database

  • PostgreSQL - RDBMS
  • Database Name - fintrack
  • Connection - DATABASE_URL
  • Port - 5432 (default)

👤 Users Table

  • id - UUID (Primary Key)
  • email - String (Unique)
  • password - String (Hashed)
  • name - String
  • Timestamps - createdAt, updatedAt

📁 Categories Table

  • id - UUID (Primary Key)
  • name - String
  • type - INCOME | EXPENSE
  • icon, color - Customization
  • isDefault - Boolean
  • userId - Foreign Key

💵 Transactions Table

  • id - UUID (Primary Key)
  • amount - Decimal(12, 2)
  • type - INCOME | EXPENSE
  • description - String (Optional)
  • date - DateTime
  • userId, categoryId - Foreign Keys
  • Index - (userId, date)

💰 Budgets Table

  • id - UUID (Primary Key)
  • amount - Decimal(12, 2)
  • month - Integer (1-12)
  • year - Integer
  • userId, categoryId - Foreign Keys
  • Unique - (userId, categoryId, month, year)

🔗 Relationships

  • User ➜ Transactions - One-to-Many
  • User ➜ Categories - One-to-Many
  • User ➜ Budgets - One-to-Many
  • Category ➜ Transactions - One-to-Many
  • Category ➜ Budgets - One-to-Many

🎯 Trách nhiệm của Data Layer

✅ Lưu trữ dữ liệu vĩnh viễn (Persistent Storage)
✅ Đảm bảo tính toàn vẹn dữ liệu (Data Integrity)
✅ Xử lý relationships giữa các bảng (Foreign Keys)
✅ Optimize queries với Indexes
✅ Backup và Recovery
🔒 CHỈ Backend (Tầng 2) mới được truy cập trực tiếp
❌ Frontend KHÔNG BAO GIỜ truy cập trực tiếp Database

✨ LỢI ÍCH CỦA KIẾN TRÚC 3 TẦNG

🎯

Separation of Concerns

Mỗi tầng có trách nhiệm rõ ràng, không chồng chéo. Frontend chỉ lo UI, Backend lo logic, Database lo lưu trữ.

🔒

Security Enhancement

Frontend không truy cập trực tiếp database. Tất cả phải qua Backend layer với authentication và authorization.

📈

Independent Scaling

Có thể scale từng tầng riêng biệt. Ví dụ: Thêm server Backend mà không ảnh hưởng Frontend hay Database.

🔧

Easy Maintenance

Dễ bảo trì và sửa lỗi. Thay đổi UI không ảnh hưởng Backend. Đổi database không ảnh hưởng Frontend.

♻️

Code Reusability

Backend API có thể tái sử dụng cho nhiều client: Web, Mobile App, Desktop App, etc.

👥

Team Collaboration

Team Frontend và Backend có thể làm việc song song độc lập. Chỉ cần thống nhất API contract.

🧪

Testing Isolation

Có thể test từng tầng riêng biệt. Unit test, Integration test, E2E test dễ dàng hơn.

🚀

Technology Flexibility

Có thể thay đổi công nghệ từng tầng. Ví dụ: Đổi React sang Vue, đổi PostgreSQL sang MySQL.
